Physics (compopt.physics)¶

compopt.physics¶

Thermo-fluid physics engine for GPU data-center simulation.

Sub-modules¶

chip

GPU chip-level RC-network thermal model (GPUChipModel, make_gpu).

electricity

Realistic grid electricity price model (GridElectricityPriceModel, TOUTariff).

fluids

Coolant loops, CDU, cooling tower, and vectorised variants.

server

Server, rack, and data-center aggregation models.

workloads

Synthetic power-profile generators for benchmarking.

Chip Model¶

compopt.physics.chip¶

GPU chip-level thermal RC-network model.

Supports H100, B200, and custom GPU configurations via parameterised dataclasses. The model represents the die as an nx × ny grid of tiles plus lumped HBM, VRM, and PCB nodes, connected to a direct-to-chip cold-plate (CoolantLoop).

Vectorised stepping is supported via ChipModelVec for batched RL.

Fluids¶

compopt.physics.fluids¶

Thermo-fluid building blocks: coolant loops, CDU, cooling towers, water models.

All classes use forward-Euler or RK4 time integration and support vectorised (batched) operation via NumPy broadcasting for parallel simulation of many identical plants (e.g. for RL vectorised envs).

class compopt.physics.fluids.CDU(UA_W_K=5000.0, T_facility_C=30.0, pump_power_W_per_kgs=500.0, hot_loop=<factory>, cold_loop=<factory>)[source]¶

Bases: object

Coolant Distribution Unit model (rack-to-facility heat exchanger).

The CDU transfers heat from the hot (rack-side) loop to the cold (facility-side) loop through a counter-flow heat exchanger characterised by an overall UA value.

Energy balance (hot side, lumped):

C_hot · dT_hot/dt = Q_rack_total − UA · (T_hot − T_cold) − ṁ_hot · c_p · (T_hot − T_hot,in)

Facility side is assumed much larger; its temperature responds slower (modelled with a separate CoolantLoop or held constant).

Parameters:

  • UA_W_K (Overall heat-transfer coefficient × area [W/K])

  • T_facility_C (Facility-side (cold) supply temperature [°C])

  • pump_power_W_per_kgs (Pump power per unit flow [W/(kg/s)])

Server / Rack / DataCenter¶

compopt.physics.server¶

Server-level and rack-level thermal models.

A ServerNode wraps one or more GPU chips, CPUs, memory, and NIC power draws into a single compute node that plugs into a rack’s coolant manifold.

A RackModel aggregates multiple servers sharing a rack coolant manifold and exposes observation / telemetry interfaces.

A DataCenterModel aggregates racks, CDUs, and a cooling tower into a full facility-level plant.
